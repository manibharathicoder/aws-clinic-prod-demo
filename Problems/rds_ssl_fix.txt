# AWS Database Connection Error — Explained & How to Fix

I have investigated the logs from Elastic Beanstalk, and here is exactly why the 500 Server Error is happening and how to fix it when you start your new chat.

## The Exact Error from the Server Logs

```
error: no pg_hba.conf entry for host "172.31.37.119", user "aim_admin", database "postgres", no encryption
```

This error comes from your AWS RDS PostgreSQL database. It means the backend (Elastic Beanstalk) reached the database successfully, but **the database rejected the connection.**

## Why did the database reject it? (Two Reasons)

1. **"no encryption":** AWS RDS databases require **SSL (secure encrypted connections)** by default. However, your Node.js backend code (`src/config/db.js`) is trying to connect without SSL. 
2. **`database "postgres"`:** Look closely at the error—it says it's trying to connect to a database named `"postgres"`. But when you created the RDS database, you named it `"aim_dev"`. Your Elastic Beanstalk environment variables have `DB_NAME=postgres`.

---

## 🛠️ The Solution (What to do in your next chat)

In your new chat, give the AI these exact steps to fix the issue:

### Step 1: Update the Elastic Beanstalk Environment Variable

Your `DB_NAME` environment variable is currently set to `postgres`. It needs to be updated to `aim_dev` (the actual name of the database you created).

Run this in the terminal:
```bash
aws elasticbeanstalk update-environment --environment-name aim-backend-dev --option-settings "Namespace=aws:elasticbeanstalk:application:environment,OptionName=DB_NAME,Value=aim_dev"
```

### Step 2: Enable SSL in the Backend Code

The AI will need to modify your database connection file to use SSL for AWS RDS.

**File:** `c:\Users\Zen\Desktop\Clinic Project\AWS Clinic Prod\aim-backend\src\config\db.js`

Change the connection code to look like this:

```javascript
const pool = new Pool({
  host: process.env.DB_HOST,
  port: process.env.DB_PORT,
  database: process.env.DB_NAME,
  user: process.env.DB_USER,
  password: process.env.DB_PASSWORD,
  // ADD THIS FOR AWS RDS SSL:
  ssl: {
    rejectUnauthorized: false
  }
});
```

### Step 3: Redeploy the Backend

Once the code is updated and the environment variable is changed, simply redeploy the backend to Elastic Beanstalk:

```bash
cd "c:\Users\Zen\Desktop\Clinic Project\AWS Clinic Prod\aim-backend"
eb deploy aim-backend-dev
```

After doing these 3 steps, the database connection will be secure and point to the correct database name (`aim_dev`). The migrations will run successfully, the `users` table will be created, and the login will finally work!
